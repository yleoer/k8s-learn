# 容器操作与排障

容器是由镜像启动出来的运行实例。本节集中学习容器启动、端口映射、资源限制、日志查看、进入容器、停止删除和清理策略。

## 启动容器

前台启动，退出后自动删除：

```bash
docker run -it --rm nginx:1.27-alpine sh
```

后台启动并映射端口：

```bash
docker run -d --name web1 -p 8080:80 nginx:1.27-alpine
```

`-p 8080:80` 表示把宿主机 `8080` 端口映射到容器内 `80` 端口。

验证：

```bash
docker ps
curl http://127.0.0.1:8080
```

## 限制容器资源

生产环境建议为容器设置资源上限，避免单个容器耗尽宿主机资源：

```bash
docker run -d --name web2 \
  --memory 256m \
  --cpus 0.5 \
  -p 8081:80 nginx:1.27-alpine
```

常用资源限制参数：

| 参数 | 作用 |
| --- | --- |
| `--memory` / `-m` | 内存上限，如 `256m`、`1g` |
| `--cpus` | CPU 核心数上限，如 `0.5`、`2` |
| `--memory-swap` | 内存 + swap 总上限 |

查看资源用量：

```bash
docker stats web2
```

这些参数与 Kubernetes Pod 中的 `resources.limits` 和 `resources.requests` 概念相关，提前理解有助于后续 K8s 资源管理。

## 查看容器

```bash
docker ps
docker ps -q
docker ps -a
```

示例输出：

```text
CONTAINER ID   IMAGE               COMMAND                  STATUS         PORTS                  NAMES
a1b2c3d4e5f6   nginx:1.27-alpine   "/docker-entrypoint.…"   Up 5 minutes   0.0.0.0:8080->80/tcp   web1
```

查看容器详细信息：

```bash
docker inspect web1
```

常看字段：

- `State.Status`：容器状态。
- `State.ExitCode`：退出码。
- `Config.Image`：使用的镜像。
- `Config.Cmd`：默认命令。
- `NetworkSettings.Networks`：网络、IP、网关。
- `Mounts`：挂载信息。
- `HostConfig.PortBindings`：端口映射。

## 查看日志

```bash
docker logs web1
docker logs -f web1
docker logs --tail 10 web1
docker logs --since 10m --tail 100 --timestamps web1
```

`docker logs` 默认读取容器标准输出和标准错误。生产镜像应把应用日志输出到 stdout/stderr，而不是只写在容器内部文件里。这样 Docker、Kubernetes、日志采集组件才能统一收集。

常用参数：

| 参数 | 作用 |
| --- | --- |
| `-f` | 持续跟随日志 |
| `--tail 100` | 只看最后 100 行 |
| `--since 10m` | 查看最近 10 分钟日志 |
| `--timestamps` | 显示时间戳 |

## 进入容器

```bash
docker exec -it web1 sh
```

如果镜像内有 bash，也可以：

```bash
docker exec -it web1 bash
```

进入容器适合临时排查，不建议在容器里手工改配置作为长期运行方式。镜像和配置应该通过 Dockerfile、挂载或环境变量管理。

常用排查命令：

```bash
docker exec web1 ps aux
docker exec web1 env
docker exec web1 ls -lah /usr/share/nginx/html
```

如果镜像非常精简，可能没有 `sh`、`ps`、`curl` 等工具。生产镜像精简是好事，排查时可以通过日志、健康检查、临时调试容器或重新构建 debug 镜像处理。

## 停止和删除容器

```bash
docker stop web1
docker kill web1
docker rm web1
docker rm -f web2
```

删除所有已退出容器：

```bash
docker container prune
docker container prune -f
```

强制删除所有容器要特别小心：

```bash
docker rm -f $(docker ps -aq)
```

PowerShell 中可以使用：

```powershell
docker ps -aq | ForEach-Object { docker rm -f $_ }
```

## 清理策略

```bash
docker system prune
docker system prune -a
```

清理前建议先确认没有需要保留的容器和镜像。

| 命令 | 会清理什么 | 风险 |
| --- | --- | --- |
| `docker container prune` | 已停止容器 | 可能删除还想保留日志的容器 |
| `docker image prune` | 悬空镜像 | 风险较低，但仍需确认 |
| `docker image prune -a` | 当前没有容器使用的镜像 | 可能导致下次启动需要重新拉取镜像 |
| `docker system prune -a` | 容器、网络、悬空构建缓存、未使用镜像 | 范围较大，不建议在生产机随手执行 |
| `docker volume prune` | 未被容器引用的卷 | 可能删除业务数据，执行前必须确认 |

## 日志大小控制

如果使用默认 `json-file` 日志驱动，长时间运行的容器可能让日志文件不断增长。可以在 `/etc/docker/daemon.json` 中限制日志大小：

```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m",
    "max-file": "3"
  }
}
```

修改后重启 Docker：

```bash
sudo systemctl restart docker
```

已存在容器不一定自动应用新日志配置，生产环境应结合发布窗口滚动重建容器。

## 本节回顾

- `docker run` 是容器启动入口，重点理解 `--name`、`-p`、`-d`、资源限制。
- `docker inspect` 能看到容器状态、网络、挂载和端口映射。
- 容器应用应优先把日志输出到 stdout/stderr。
- `docker exec` 适合临时排查，不应把手工修改容器作为发布方式。
- `prune` 类命令需要先理解删除范围，尤其要谨慎处理 volume。
