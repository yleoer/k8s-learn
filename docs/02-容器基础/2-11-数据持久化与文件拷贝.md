# 数据持久化与文件拷贝

容器默认文件系统是临时的。容器删除后，可写层也会删除。因此需要用挂载或卷保存重要数据。

## 文件挂载

把宿主机文件挂载到容器内：

```bash
docker run -it --rm -v /etc/hosts:/etc/hosts nginx:alpine sh
cat /etc/hosts
```

如果容器内原本存在同名文件，会被宿主机文件覆盖。

## 目录挂载

```bash
mkdir -p ./html
echo 'hello nginx' > ./html/index.html
docker run -d --name web -p 8080:80 -v "$PWD/html:/usr/share/nginx/html" nginx:alpine
curl http://127.0.0.1:8080
```

目录挂载常用于配置文件、静态资源、日志目录和本地开发代码目录。

## Docker volume

Docker volume 由 Docker 管理，适合保存应用数据：

```bash
docker volume create nginx-data
docker run -d --name web -v nginx-data:/usr/share/nginx/html nginx:alpine
docker volume ls
docker volume inspect nginx-data
```

## 文件拷贝

从宿主机复制到容器：

```bash
docker cp install.sh <container-id>:/tmp/install.sh
```

从容器复制到宿主机：

```bash
docker cp <container-id>:/var/log/nginx/access.log ./access.log
```

## 注意事项

- 挂载文件或目录会覆盖容器内原路径内容。
- Linux 下要注意文件属主和权限。
- Windows 或 macOS 下要注意路径格式和文件共享设置。
- 生产数据优先使用 volume 或明确规划的宿主机目录。
