# 持久化与服务部署

容器默认文件系统是临时的。容器删除后，可写层也会删除。因此需要用挂载或卷保存重要数据。部署一个 Docker 服务时，也不建议只记一条 `docker run` 命令，而应该形成固定检查流程。

## 数据类型与处理方式

| 类型 | 示例 | 推荐处理方式 |
| --- | --- | --- |
| 镜像内置文件 | 应用二进制、默认配置 | 写入 Dockerfile，随镜像发布 |
| 运行时配置 | nginx 配置、应用配置文件 | 使用 bind mount、环境变量或配置中心 |
| 业务数据 | 数据库文件、上传文件 | 使用 Docker volume 或明确规划的数据目录 |

## 文件和目录挂载

把宿主机文件挂载到容器内：

```bash
docker run -it --rm -v /etc/hosts:/etc/hosts nginx:1.27-alpine sh
cat /etc/hosts
```

如果容器内原本存在同名文件，会被宿主机文件覆盖。

目录挂载：

```bash
mkdir -p ./html
echo 'hello nginx' > ./html/index.html
docker run -d --name web -p 8080:80 -v "$PWD/html:/usr/share/nginx/html" nginx:1.27-alpine
curl http://127.0.0.1:8080
```

验证挂载是否生效：

```bash
docker inspect web --format '{{json .Mounts}}'
```

如果页面没有变化，优先检查：

- 宿主机路径是否写对。
- 容器内路径是否是应用实际读取路径。
- 文件权限是否允许容器进程读取。
- Windows/macOS Docker Desktop 是否允许共享该目录。

## Docker volume

Docker volume 由 Docker 管理，适合保存应用数据：

```bash
docker volume create nginx-data
docker run -d --name web-volume -v nginx-data:/usr/share/nginx/html nginx:1.27-alpine
docker volume ls
docker volume inspect nginx-data
```

bind mount 与 volume 的选择：

| 方式 | 示例 | 适合场景 |
| --- | --- | --- |
| bind mount | `-v "$PWD/html:/usr/share/nginx/html"` | 本地开发、挂载配置、明确指定宿主机目录 |
| named volume | `-v nginx-data:/usr/share/nginx/html` | 保存运行数据，让 Docker 管理数据目录 |
| anonymous volume | `-v /var/lib/mysql` | 临时实验，不建议生产长期依赖 |

删除容器不会自动删除 named volume，但 `docker volume rm`、`docker volume prune`、`docker compose down -v` 会删除卷数据。

## 文件拷贝

从宿主机复制到容器：

```bash
docker cp install.sh <container-id>:/tmp/install.sh
```

从容器复制到宿主机：

```bash
docker cp <container-id>:/var/log/nginx/access.log ./access.log
```

`docker cp` 适合临时拷贝文件和排查，不应作为长期配置分发方式。长期配置应使用镜像、挂载、环境变量或配置中心。

## 备份与恢复 volume

备份 volume 数据可以使用临时容器挂载同一个卷：

```bash
docker run --rm \
  -v nginx-data:/data \
  -v "$PWD/backup:/backup" \
  alpine \
  tar czf /backup/nginx-data.tgz -C /data .
```

恢复到新 volume：

```bash
docker volume create nginx-data-restore
docker run --rm \
  -v nginx-data-restore:/data \
  -v "$PWD/backup:/backup" \
  alpine \
  tar xzf /backup/nginx-data.tgz -C /data
```

验证：

```bash
docker run --rm -v nginx-data-restore:/data alpine ls -lah /data
```

生产环境中，数据库类服务应优先使用数据库自身的备份工具，例如 `mysqldump`、`pg_dump`、物理备份或存储快照，而不是只打包数据目录。

## 服务部署通用步骤

部署前先明确：

- 容器监听端口是什么。
- 宿主机要暴露哪个端口。
- 是否需要挂载配置文件。
- 是否需要持久化数据。
- 是否需要环境变量。
- 是否需要重启策略。
- 内存和 CPU 上限是多少。

确认镜像：

```bash
docker search nginx
docker pull nginx:1.27-alpine
docker images | grep nginx
```

启动容器：

```bash
docker run -d \
  --name nginx-demo \
  --restart unless-stopped \
  --memory 256m \
  --cpus 0.5 \
  -p 8080:80 \
  -v "$PWD/html:/usr/share/nginx/html" \
  nginx:1.27-alpine
```

验证服务：

```bash
docker ps
docker logs --tail 50 nginx-demo
curl http://127.0.0.1:8080
```

排查问题：

```bash
docker inspect nginx-demo
docker exec -it nginx-demo sh
sudo ss -lntup | grep 8080
```

常见问题：端口被占用、挂载路径错误、容器内应用启动失败、镜像架构不匹配、文件权限不足。

清理服务：

```bash
docker stop nginx-demo
docker rm nginx-demo
docker rmi nginx:1.27-alpine
```
