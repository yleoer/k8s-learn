# 镜像大小优化

镜像越小，拉取越快，漏洞面越小，分发成本越低。优化镜像大小不是盲目追求极限，而是在可维护性和体积之间取得平衡。

## 选择合适基础镜像

常见选择：

- `alpine`：体积小，适合静态编译或依赖简单的应用。
- `debian:slim`：兼容性更好，体积适中。
- distroless：运行时极简，适合成熟生产镜像。

注意 Alpine 使用 musl libc，部分依赖 glibc 的程序可能需要额外处理。

## 多阶段构建

Go 应用示例：

```dockerfile
FROM golang:1.23-alpine AS builder
WORKDIR /src
COPY . .
RUN go build -o app .

FROM alpine:3.20
WORKDIR /app
COPY --from=builder /src/app /app/app
CMD ["/app/app"]
```

构建工具和源码只留在 builder 阶段，最终镜像只保留运行所需文件。

## 合并和清理指令

```dockerfile
RUN apt-get update \
    && apt-get install -y --no-install-recommends ca-certificates curl \
    && rm -rf /var/lib/apt/lists/*
```

重点：安装和清理要在同一个 `RUN` 中完成。

## 减少构建上下文

使用 `.dockerignore` 排除无关文件：

```text
.git
node_modules
*.log
dist
.tmp
```

## 优化建议清单

- 使用明确且较小的基础镜像。
- 多阶段构建，只保留运行时产物。
- 合并安装和清理步骤。
- 不把构建缓存、包管理器缓存留进镜像。
- 使用 `.dockerignore`。
- 删除不必要的调试工具和临时文件。
