# 业务镜像实战：前端、PHP 与 Go 后端

前几节分别讲解了各条指令和优化方法，本节用三个完整的生产级范例把前面的知识串起来：一个前端静态站点、一个 PHP Web 应用、一个 Go 后端微服务。

## 制作前需要明确的问题

每做一个业务镜像之前，先回答：

- 应用启动命令是什么？监听端口是什么？
- 配置来自文件、环境变量还是配置中心？
- 是否需要非 root 用户运行？是否需要健康检查？
- 是否需要持久化数据？日志输出到哪里？
- 内存和 CPU 上限大概是多少？

## 范例一：前端静态站点

### 项目结构

```text
frontend/
├── Dockerfile
├── .dockerignore
└── src/
    ├── index.html
    ├── style.css
    └── app.js
```

### .dockerignore

```text
.git
node_modules
*.log
.DS_Store
README.md
```

### 静态文件

`src/index.html`：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head><meta charset="UTF-8"><title>K8s 学习平台</title>
<link rel="stylesheet" href="style.css"></head>
<body>
  <div class="container"><h1>Kubernetes 学习平台</h1>
  <p>版本：<span id="version"></span></p></div>
  <script src="app.js"></script>
</body></html>
```

`src/style.css`：

```css
body{font-family:system-ui,sans-serif;display:flex;justify-content:center;
align-items:center;min-height:100vh;margin:0;background:#f5f5f5}
.container{text-align:center;padding:2rem;background:#fff;border-radius:8px;
box-shadow:0 2px 8px rgba(0,0,0,.1)}
```

`src/app.js`：

```javascript
document.getElementById('version').textContent = 'v1.0.0';
```

### Dockerfile

```dockerfile
FROM nginx:1.27-alpine
LABEL maintainer="platform@example.com"
LABEL org.opencontainers.image.title="frontend"
LABEL org.opencontainers.image.version="1.0.0"

COPY src/ /usr/share/nginx/html/

HEALTHCHECK --interval=30s --timeout=3s --retries=3 \
  CMD wget -qO- http://127.0.0.1:80/ || exit 1

EXPOSE 80
```

> `nginx:alpine` 内置 `wget`，可直接用于健康检查。nginx worker 进程自动以非 root 运行。

### 构建并验证

```bash
docker build -t registry.example.com/team/frontend:v1.0.0 .
```

```bash

  --memory 128m --cpus 0.25 \
  -p 8080:80 \
  registry.example.com/team/frontend:v1.0.0

curl http://127.0.0.1:8080
docker ps --filter name=frontend
```

## 范例二：PHP Web 应用

### 项目结构

```text
php-app/
├── Dockerfile
├── .dockerignore
├── composer.json
└── public/
    └── index.php
```

### .dockerignore

```text
.git
*.log
.DS_Store
README.md
vendor/
```

### composer.json

```json
{
  "name": "example/php-app",
  "description": "A simple PHP web application",
  "require": {
    "php": ">=8.3",
    "monolog/monolog": "^3.0"
  },
  "autoload": {
    "psr-4": {
      "App\\": "src/"
    }
  }
}
```

### public/index.php

```php
<?php
require_once __DIR__ . '/../vendor/autoload.php';

use Monolog\Logger;
use Monolog\Handler\StreamHandler;

$log = new Logger('app');
$log->pushHandler(new StreamHandler('php://stdout', Logger::INFO));

$uri = parse_url($_SERVER['REQUEST_URI'], PHP_URL_PATH);

header('Content-Type: application/json');

if ($uri === '/healthz') {
    echo json_encode([
        'status'  => 'ok',
        'version' => getenv('APP_VERSION') ?: '1.0.0',
        'time'    => date('c'),
    ]);
    exit;
}

if ($uri === '/api/hello') {
    $log->info('hello endpoint called');
    echo json_encode(['message' => 'Hello from PHP on K8s!']);
    exit;
}

http_response_code(404);
echo json_encode(['error' => 'not found']);
```

### Dockerfile

```dockerfile
FROM composer:2 AS vendor
WORKDIR /app
COPY composer.json ./
RUN composer install --no-dev --no-interaction --prefer-dist

FROM php:8.3-apache
LABEL maintainer="platform@example.com"
LABEL org.opencontainers.image.title="php-app"
LABEL org.opencontainers.image.version="1.0.0"

RUN apt-get update \
    && apt-get install -y --no-install-recommends wget \
    && rm -rf /var/lib/apt/lists/*

RUN docker-php-ext-install opcache

ENV APP_VERSION=1.0.0

COPY --from=vendor /app/vendor/ /var/www/html/vendor/
COPY public/ /var/www/html/

RUN chown -R www-data:www-data /var/www/html

HEALTHCHECK --interval=30s --timeout=3s --retries=3 \
  CMD wget -qO- http://127.0.0.1:80/healthz || exit 1

EXPOSE 80
```

关键点：

- **多阶段构建**：第一阶段用 `composer:2` 安装依赖，最终镜像不包含 Composer 本身。
- **基础镜像**：`php:8.3-apache` 同时运行 Apache 和 PHP，适合多数场景。高流量场景可改用 `php:8.3-fpm` + 独立 nginx。
- **opcache**：启用 PHP 字节码缓存，提升运行性能。
- **非 root**：Apache 以 `www-data` 用户运行，`chown` 确保文件可读。
- **日志输出**：Monolog 写入 `php://stdout`，由 Docker 收集。

### 构建并验证

```bash
docker build -t registry.example.com/team/php-app:v1.0.0 .
```

```bash
docker run -d --name php-app \
  --memory 256m --cpus 0.5 \
  -p 8080:80 \
  registry.example.com/team/php-app:v1.0.0

curl http://127.0.0.1:8080/healthz
curl http://127.0.0.1:8080/api/hello
```

## 范例三：Go 后端微服务

### 项目结构

```text
backend/
├── Dockerfile
├── .dockerignore
├── go.mod
├── go.sum
└── main.go
```

### .dockerignore

```text
.git
*.log
.DS_Store
README.md
```

### main.go

```go
package main

import (
	"encoding/json"
	"log"
	"net/http"
	"os"
	"time"
)

func healthHandler(w http.ResponseWriter, r *http.Request) {
	w.Header().Set("Content-Type", "application/json")
	json.NewEncoder(w).Encode(map[string]string{
		"status":    "ok",
		"version":   getEnv("APP_VERSION", "1.0.0"),
		"timestamp": time.Now().UTC().Format(time.RFC3339),
	})
}

func helloHandler(w http.ResponseWriter, r *http.Request) {
	w.Header().Set("Content-Type", "application/json")
	json.NewEncoder(w).Encode(map[string]string{
		"message": "Hello from K8s!",
	})
}

func getEnv(key, fallback string) string {
	if v := os.Getenv(key); v != "" { return v }
	return fallback
}

func main() {
	port := getEnv("APP_PORT", "8080")
	mux := http.NewServeMux()
	mux.HandleFunc("/healthz", healthHandler)
	mux.HandleFunc("/api/hello", helloHandler)
	log.Printf("server starting on :%s", port)
	log.Fatal(http.ListenAndServe(":"+port, mux))
}
```

### go.mod

```text
module github.com/example/backend

go 1.23
```

### Dockerfile

```dockerfile
FROM golang:1.23-alpine AS builder
WORKDIR /src
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 go build -ldflags="-s -w" -o /app/server .

FROM alpine:3.20
LABEL maintainer="platform@example.com"
LABEL org.opencontainers.image.title="backend"
LABEL org.opencontainers.image.version="1.0.0"

RUN apk add --no-cache ca-certificates tzdata wget

ENV APP_PORT=8080 \
    APP_VERSION=1.0.0

RUN addgroup -S app && adduser -S app -G app
WORKDIR /app
COPY --from=builder --chown=app:app /app/server /app/server

USER app
EXPOSE 8080

HEALTHCHECK --interval=15s --timeout=3s --retries=3 \
  CMD wget -qO- http://127.0.0.1:8080/healthz || exit 1

CMD ["/app/server"]
```

关键点：多阶段构建 → 静态编译 → 非 root 用户 → HEALTHCHECK → 添加 ca-certificates、tzdata 和 wget。

### 构建并验证

```bash

```

```bash

  registry.example.com/team/backend:v1.0.0

curl http://127.0.0.1:8080/healthz
curl http://127.0.0.1:8080/api/hello
```

## 三个范例对照

| 对比项 | 前端静态站点 | PHP Web 应用 | Go 后端 |
| --- | --- | --- | --- |
| 基础镜像 | `nginx:1.27-alpine` | `php:8.3-apache` | `alpine:3.20` |
| 构建方式 | 单阶段 | 多阶段（composer → php） | 多阶段（golang → alpine） |
| 非 root | nginx 自动处理 | Apache www-data | `USER app` |
| 健康检查 | `wget` 首页 | `wget /healthz` | `wget /healthz` |
| 配置注入 | 构建时 COPY | 环境变量 | 环境变量 |
| 最终体积 | ~45 MB | ~180 MB | ~19 MB |

## 推送与部署

```bash
docker push registry.example.com/team/frontend:v1.0.0
docker push registry.example.com/team/php-app:v1.0.0
docker push registry.example.com/team/backend:v1.0.0
```

推送后在 K8s 中部署：

```bash
kubectl create deployment frontend --image=registry.example.com/team/frontend:v1.0.0
kubectl create deployment backend --image=registry.example.com/team/backend:v1.0.0
kubectl expose deployment backend --port=8080
```

Dockerfile 中的 `HEALTHCHECK` 只影响 Docker 对容器健康状态的判断。部署到 Kubernetes 时，建议在 Deployment 中显式配置探针：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: backend
spec:
  replicas: 2
  selector:
    matchLabels:
      app: backend
  template:
    metadata:
      labels:
        app: backend
    spec:
      containers:
        - name: backend
          image: registry.example.com/team/backend:v1.0.0
          ports:
            - containerPort: 8080
          readinessProbe:
            httpGet:
              path: /healthz
              port: 8080
            initialDelaySeconds: 5
            periodSeconds: 10
          livenessProbe:
            httpGet:
              path: /healthz
              port: 8080
            initialDelaySeconds: 15
            periodSeconds: 20
          resources:
            requests:
              cpu: 100m
              memory: 128Mi
            limits:
              cpu: 500m
              memory: 256Mi
```

## 发布前检查清单

- 镜像 tag 是否明确（不是 `latest`）。
- 镜像是否能在目标架构运行。
- 应用是否非 root 运行。
- 镜像中是否包含敏感信息。
- 是否配置了 `.dockerignore`。
- 是否配置了 HEALTHCHECK 且端点可用。
- 日志是否输出到 stdout/stderr。
- 镜像大小是否合理。

## 本章回顾

完成本章后，你应该具备以下能力：

- 理解 Dockerfile 的常用指令及其组合方式。
- 理解 Shell 格式与 Exec 格式的区别，能正确选择。
- 理解镜像分层原理和构建缓存策略，优化指令顺序。
- 掌握 CMD 与 ENTRYPOINT 的配合、ARG 与 ENV 的区别。
- 使用多阶段构建分离构建和运行环境，减小镜像体积。
- 使用 `.dockerignore` 排除无关文件。
- 使用 `docker buildx` 构建多架构镜像。
- 按标准流程为非 root 用户制作带健康检查的生产级镜像。
- 独立完成前端静态站点和 Go 后端微服务的镜像制作与发布。
