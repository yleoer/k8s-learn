# 业务镜像制作标准流程

业务镜像制作的目标不是“能跑就行”，而是稳定、可复现、可发布、可排查。

## 1. 明确应用运行方式

先回答这些问题：

- 应用启动命令是什么？
- 监听端口是什么？
- 配置来自文件、环境变量还是配置中心？
- 是否需要写日志到文件？
- 是否需要非 root 用户运行？
- 是否需要健康检查？

## 2. 选择基础镜像

按应用类型选择：

- 前端静态站点：`nginx:alpine`
- Java 应用：JRE 或 distroless Java 镜像
- Go 应用：多阶段构建 + Alpine 或 distroless
- Node.js 应用：`node:<version>-alpine` 或 slim

## 3. 编写 Dockerfile

通用模板：

```dockerfile
FROM alpine:3.20
LABEL maintainer="platform@example.com"

WORKDIR /app
COPY --chown=1001:1001 app /app/app

USER 1001
EXPOSE 8080
CMD ["/app/app"]
```

## 4. 构建并打 tag

```bash
docker build -t registry.example.com/team/app:v1.0.0 .
```

建议 tag 包含应用版本、Git commit、环境标识或构建号。

## 5. 本地验证

```bash
docker run --rm -p 8080:8080 registry.example.com/team/app:v1.0.0
curl http://127.0.0.1:8080/healthz
```

检查：

```bash
docker logs <container>
docker inspect <container>
docker exec -it <container> sh
```

## 6. 推送镜像

```bash
docker push registry.example.com/team/app:v1.0.0
```

## 7. 发布前检查

- 镜像 tag 是否明确。
- 镜像是否能在目标架构运行。
- 应用是否非 root 运行。
- 镜像中是否包含敏感信息。
- 镜像大小是否合理。
- 健康检查是否可用。
- 日志是否输出到 stdout/stderr。
