# 环境规划与版本选择

环境准备的第一步是定版本、定网段、定节点角色。后续 kubeadm 初始化、Calico 安装、worker 节点加入和排障都会依赖这些信息。

## 默认环境

| 项目 | 选择 |
| --- | --- |
| 操作系统 | Ubuntu 24.04 LTS，代号 `noble` |
| Kubernetes | v1.36.x |
| 容器运行时 | containerd |
| CNI 网络插件 | Calico |
| 安装方式 | kubeadm |
| Pod 网段 | `10.244.0.0/16` |
| Service 网段 | `10.96.0.0/12` |

Kubernetes APT 源按小版本分仓库，例如 `v1.36`、`v1.35`。安装 `kubelet`、`kubeadm`、`kubectl` 时建议使用同一个小版本仓库。

## 节点规划

| 角色 | 主机名 | 示例 IP | 说明 |
| --- | --- | --- | --- |
| control-plane | `master` | `192.168.2.108` | 控制平面节点 |
| worker | `worker01` | `192.168.2.109` | 工作节点 |
| worker | `worker02` | `192.168.2.110` | 工作节点 |

每台节点建议至少 2 核 CPU、2 GiB 内存、30 GiB 磁盘，并保证节点之间网络互通、主机名和 IP 稳定、系统时间同步。实验环境可以先用单 control-plane 加 worker 节点跑通，再扩展到高可用架构。

## 主机名和解析

设置主机名：

```bash
sudo hostnamectl set-hostname master
```

如果没有内部 DNS，可以在所有节点写入 `/etc/hosts`：

```bash
sudo tee -a /etc/hosts >/dev/null <<'EOF'
192.168.2.108 master
192.168.2.109 worker01
192.168.2.110 worker02
EOF
```

检查网络和主机名解析：

```bash
hostname
hostname -I
ping -c 3 worker01
ping -c 3 worker02
```

## 规划原则

- Pod CIDR 必须和 Calico 的 IPPool CIDR 保持一致。
- Kubernetes 节点通信建议优先使用稳定内网 IP。
