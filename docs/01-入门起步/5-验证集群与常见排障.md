# 验证集群与常见排障

本节通过 nginx Deployment 和 NodePort Service 验证集群调度、Pod 网络和 Service 转发能力，并整理初始化阶段最常用的排障命令。

## 部署测试应用

```bash
kubectl create deployment nginx --image=nginx:1.27
kubectl expose deployment nginx --port=80 --type=NodePort
```

查看状态：

```bash
kubectl get deploy,pod,svc -o wide
```

## 访问测试

获取 NodePort：

```bash
NODE_PORT=$(kubectl get svc nginx -o jsonpath='{.spec.ports[0].nodePort}')
curl "http://127.0.0.1:${NODE_PORT}"
```

如果从集群外访问，使用任意节点 IP：

```bash
curl "http://<node-ip>:${NODE_PORT}"
```

清理资源：

```bash
kubectl delete svc nginx
kubectl delete deployment nginx
```

## 常用检查命令

节点和 Pod：

```bash
kubectl get nodes -o wide
kubectl get pods -A -o wide
kubectl describe node <node-name>
```

控制平面：

```bash
kubectl get --raw='/readyz?verbose'
kubectl get --raw='/livez?verbose'
kubectl -n kube-system get pods -o wide
kubectl -n kube-system logs -l component=kube-apiserver
```

containerd：

```bash
sudo systemctl status containerd --no-pager
sudo crictl info
sudo crictl ps -a
sudo crictl images
```

kubelet：

```bash
sudo systemctl status kubelet --no-pager
sudo journalctl -u kubelet -xe --no-pager
```

网络和 Calico：

```bash
ip addr
ip route
sysctl net.ipv4.ip_forward
lsmod | grep br_netfilter
kubectl get pods -n calico-system -o wide
kubectl get installation default -o yaml
kubectl get ippool -A -o yaml
```

## 高频问题

### kubeadm init 提示 swap 未关闭

```bash
swapon --show
cat /etc/fstab | grep -i swap
```

处理：

```bash
sudo swapoff -a
sudo vim /etc/fstab
```

注释 swap 行后重新执行 kubeadm。

### 节点一直 NotReady

常见原因包括 CNI 没有安装或安装失败、`--pod-network-cidr` 和 Calico IPPool CIDR 不一致、containerd 未正常运行、kubelet 异常。

```bash
kubectl get pods -A -o wide
kubectl describe node <node-name>
sudo systemctl status containerd --no-pager
sudo systemctl status kubelet --no-pager
sudo journalctl -u kubelet -xe --no-pager
```

### 镜像拉取失败

检查 containerd 是否启用了 registry hosts 配置：

```bash
grep -n "config_path" -A 3 /etc/containerd/config.toml
sudo systemctl restart containerd
sudo crictl pull registry.k8s.io/pause:3.10.2
sudo crictl pull docker.io/library/nginx:1.27
```

如果仍失败，换用可访问的镜像源或检查 DNS、代理和防火墙。

### kubelet cgroup driver 不匹配

Kubernetes 推荐 kubelet 和容器运行时都使用 systemd cgroup driver。检查 containerd：

```bash
grep -n "SystemdCgroup" /etc/containerd/config.toml
```

应为：

```text
SystemdCgroup = true
```

修改后重启：

```bash
sudo systemctl restart containerd
sudo systemctl restart kubelet
```

### 访问测试应用失败

```bash
kubectl get deploy,pod,svc -o wide
kubectl describe pod -l app=nginx
kubectl describe svc nginx
kubectl get nodes -o wide
kubectl get pods -A -o wide
sudo ss -lntup
```

常见原因包括 Pod 镜像拉取失败、节点未 Ready、Calico 未正常运行、防火墙未放通 NodePort、从外部访问时使用了错误的节点 IP。

## 重置实验集群

如果是实验环境，需要彻底重来，可以在相关节点执行：

```bash
sudo kubeadm reset
sudo rm -rf /etc/cni/net.d
sudo rm -rf "$HOME/.kube"
sudo systemctl restart containerd
```

该操作会清理当前节点上的 Kubernetes 配置。生产环境不要直接执行。

## 本章回顾

完成本章后，你应该具备以下能力：

- 根据节点角色和网段规划一套 K8s 实验集群。

- 将 worker 节点加入集群，验证节点 Ready 状态。
- 部署测试应用并通过 NodePort 访问。
- 使用常用命令排查节点、Pod、CNI、镜像和 kubelet 问题。
