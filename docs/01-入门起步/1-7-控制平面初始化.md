# 控制平面初始化

本节仅在第一个 control-plane 节点执行。

## 查看并拉取镜像

先查看 kubeadm 需要的镜像：

```bash
sudo kubeadm config images list
```

可以提前拉取镜像：

```bash
sudo kubeadm config images pull
```

如果访问 `registry.k8s.io` 较慢，可以指定镜像仓库：

```bash
sudo kubeadm config images pull \
  --image-repository registry.aliyuncs.com/google_containers
```

## 初始化集群

```bash
sudo kubeadm init \
  --kubernetes-version v1.36.1 \
  --image-repository registry.aliyuncs.com/google_containers \
  --pod-network-cidr 10.244.0.0/16 \
  --service-cidr 10.96.0.0/12 \
  --cri-socket unix:///run/containerd/containerd.sock
```

参数说明：

- `--kubernetes-version`：指定 Kubernetes 版本，建议与安装的 kubeadm 小版本一致。
- `--image-repository`：指定控制平面镜像仓库，国内网络常用阿里云镜像。
- `--pod-network-cidr`：Pod 网段，必须和后续 Calico IPPool CIDR 一致。
- `--service-cidr`：Service 网段。
- `--cri-socket`：指定 containerd 的 CRI socket。

如果 kubeadm 初始化成功，会输出 `kubeadm join ...` 命令。保存这段命令，worker 节点加入集群时需要使用。

## 配置 kubectl

```bash
mkdir -p "$HOME/.kube"
sudo cp -i /etc/kubernetes/admin.conf "$HOME/.kube/config"
sudo chown "$(id -u):$(id -g)" "$HOME/.kube/config"
```

检查节点和系统 Pod：

```bash
kubectl get nodes -o wide
kubectl get pods -A
```

此时节点通常是 `NotReady`，因为 CNI 网络插件还没有安装。
