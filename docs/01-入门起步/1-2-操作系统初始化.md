# 操作系统初始化

本节完成 Ubuntu 24.04 的基础更新，并规划虚拟机快照点。快照能显著降低实验环境反复重装的成本。

## 系统更新

所有节点执行：

```bash
sudo apt update
sudo apt full-upgrade -y
sudo apt autoremove -y
sudo reboot
```

重启后检查内核版本：

```bash
uname -r
```

`apt upgrade` 和 `apt full-upgrade` 的区别：

- `apt upgrade`：更保守，不会主动删除已有软件包。
- `apt full-upgrade`：可以根据依赖关系安装或删除软件包，系统升级更彻底。

## 建议快照点

| 快照点 | 时机 | 用途 |
| --- | --- | --- |
| `os-fresh` | Ubuntu 安装完成后 | 回到纯净系统 |
| `os-updated` | 系统更新和重启后 | 避免重复下载系统更新 |
| `base-tools` | 基础工具、主机名、网络配置完成后 | Runtime 安装失败时回滚 |
| `before-kubeadm` | containerd、kubelet、kubeadm 安装完成后 | kubeadm 初始化失败时重试 |
| `cluster-ready` | Calico 安装完成且节点 Ready 后 | 保留可用集群基线 |

## 基础检查

```bash
hostname
hostname -I
date
timedatectl
ip addr
ip route
```

确认节点之间网络互通：

```bash
ping -c 3 <node-ip>
```

如果这里的主机名、IP 或时间不稳定，先修正再继续。Kubernetes 对证书、节点身份和组件通信都比较敏感，基础环境越干净，后面排障越轻松。
