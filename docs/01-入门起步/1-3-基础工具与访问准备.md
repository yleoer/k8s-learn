# 基础工具与访问准备

本节安装 Kubernetes 环境准备常用工具，并处理 sudo、远程访问和防火墙策略。

## 安装基础工具

所有节点执行：

```bash
sudo apt update
sudo apt install -y \
  apt-transport-https \
  ca-certificates \
  curl \
  gpg \
  gnupg \
  lsb-release \
  vim \
  bash-completion \
  socat \
  conntrack \
  ipset \
  ipvsadm
```

## 配置 sudo 免密码

个人实验环境可以为当前用户配置 sudo 免密码。生产环境要按安全规范评估后再使用。

先设置 `visudo` 默认编辑器：

```bash
sudo update-alternatives --config editor
```

以下示例用户为 `yleoer`，请替换成实际用户名：

```bash
sudo visudo -f /etc/sudoers.d/yleoer
```

写入：

```text
yleoer ALL=(ALL:ALL) NOPASSWD: ALL
```

设置权限并测试：

```bash
sudo chmod 440 /etc/sudoers.d/yleoer
sudo -k
sudo whoami
```

如果输出 `root`，说明配置成功。

## 可选：安装 Tailscale

跨网络访问或远程管理节点时，可以安装 Tailscale：

```bash
sudo apt update
sudo apt install -y curl gnupg

curl -fsSL https://pkgs.tailscale.com/stable/ubuntu/noble.noarmor.gpg | \
  sudo tee /usr/share/keyrings/tailscale-archive-keyring.gpg >/dev/null

curl -fsSL https://pkgs.tailscale.com/stable/ubuntu/noble.tailscale-keyring.list | \
  sudo tee /etc/apt/sources.list.d/tailscale.list

sudo apt update
sudo apt install -y tailscale
sudo tailscale up
```

检查状态：

```bash
tailscale status
ip addr show tailscale0
```

## 防火墙策略

实验环境可以关闭 UFW，减少网络排障变量：

```bash
sudo ufw status
sudo ufw disable
sudo systemctl disable --now ufw
sudo systemctl status ufw
```

生产环境不建议简单关闭防火墙，应按需放通端口。

控制平面常用端口：

| 协议 | 端口 | 用途 |
| --- | --- | --- |
| TCP | 6443 | Kubernetes API Server |
| TCP | 2379-2380 | etcd |
| TCP | 10250 | kubelet API |
| TCP | 10257 | kube-controller-manager |
| TCP | 10259 | kube-scheduler |

工作节点常用端口：

| 协议 | 端口 | 用途 |
| --- | --- | --- |
| TCP | 10250 | kubelet API |
| TCP | 30000-32767 | NodePort Service |

Calico VXLAN 常见端口：

| 协议 | 端口 | 用途 |
| --- | --- | --- |
| UDP | 4789 | VXLAN |
| TCP | 179 | BGP，仅 BGP 模式需要 |
| IP Protocol | 4 | IP-in-IP，仅 IPIP 模式需要 |
