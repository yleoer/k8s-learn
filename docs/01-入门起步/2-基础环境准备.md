# 基础环境准备

本节在所有节点执行，目标是完成系统更新、基础工具、远程访问策略、内核网络和 swap 配置，让 Linux 环境满足 Kubernetes 与 containerd 的运行要求。

## 系统更新

```bash
sudo apt update
sudo apt full-upgrade -y
sudo apt autoremove -y
sudo reboot
```

重启后检查：

```bash
uname -r
hostname
hostname -I
date
timedatectl
ip addr
ip route
```

设置为国内常用时区：

```bash
sudo timedatectl set-timezone Asia/Shanghai
timedatectl
```

如果主机名、IP 或时间不稳定，先修正再继续。Kubernetes 对证书、节点身份和组件通信都比较敏感，基础环境越干净，后面排障越轻松。

## 安装基础工具

```bash
sudo apt update
sudo apt install -y \
  apt-transport-https \
  ca-certificates \
  curl \
  gpg \
  gnupg \
  lsb-release \
  vim \
  bash-completion \
  socat \
  conntrack \
  ipset \
  ipvsadm
```

个人实验环境可以配置 sudo 免密码。生产环境要按安全规范评估后再使用：

先把 `visudo` 默认编辑器切换为 Vim：

```bash
sudo update-alternatives --config editor
```

在列表中选择 `/usr/bin/vim.basic` 对应的编号。如果系统没有 Vim，先安装：

```bash
sudo apt install -y vim
```

然后编辑 sudoers 片段：

```bash
sudo visudo -f /etc/sudoers.d/<user>
```

写入：

```text
<user> ALL=(ALL:ALL) NOPASSWD: ALL
```

设置权限并测试：

```bash
sudo chmod 440 /etc/sudoers.d/<user>
sudo -k
sudo whoami
```

## 可选：远程访问与防火墙

跨网络访问或远程管理节点时，可以安装 Tailscale。若使用 Ubuntu 24.04，请使用 `noble` 源：

```bash
curl -fsSL https://pkgs.tailscale.com/stable/ubuntu/noble.noarmor.gpg | \
  sudo tee /usr/share/keyrings/tailscale-archive-keyring.gpg >/dev/null

curl -fsSL https://pkgs.tailscale.com/stable/ubuntu/noble.tailscale-keyring.list | \
  sudo tee /etc/apt/sources.list.d/tailscale.list

sudo apt update
sudo apt install -y tailscale
sudo tailscale up
```

实验环境可以关闭 UFW，减少网络排障变量：

```bash
sudo ufw status
sudo ufw disable
sudo systemctl disable --now ufw
```

生产环境不要简单关闭防火墙，应按节点角色放通必要端口。以下以 UFW 为例。

control-plane 节点放通 API Server、etcd 和控制面组件端口：

```bash
sudo ufw allow 6443/tcp
sudo ufw allow 2379:2380/tcp
sudo ufw allow 10250/tcp
sudo ufw allow 10257/tcp
sudo ufw allow 10259/tcp
```

worker 节点放通 kubelet 和 NodePort 端口：

```bash
sudo ufw allow 10250/tcp
sudo ufw allow 30000:32767/tcp
```

如果使用 Calico VXLAN 模式，所有节点放通 VXLAN 端口：

```bash
sudo ufw allow 4789/udp
```

如果使用 Calico BGP 模式，所有参与 BGP 的节点放通：

```bash
sudo ufw allow 179/tcp
```

启用并检查规则：

```bash
sudo ufw enable
sudo ufw status numbered
```

如果节点之间还有云安全组、硬件防火墙或上游 ACL，也要同步放通相同端口。

## 关闭 swap

立即关闭：

```bash
sudo swapoff -a
swapon --show
```

永久禁用：

```bash
sudo vim /etc/fstab
```

注释 swap 行，例如：

```text
# /swap.img none swap sw 0 0
```

验证 `/etc/fstab` 没有语法错误：

```bash
sudo mount -a
```

关闭 swap 是为了让 kubelet 能根据真实内存压力做资源管理和驱逐判断，避免 Pod 在内存不足时表现不可预测。

## 配置内核模块和 sysctl

```bash
sudo tee /etc/modules-load.d/k8s.conf >/dev/null <<'EOF'
overlay
br_netfilter
EOF

sudo modprobe overlay
sudo modprobe br_netfilter
```

配置网络转发：

```bash
sudo tee /etc/sysctl.d/k8s.conf >/dev/null <<'EOF'
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF

sudo sysctl --system
```

检查结果：

```bash
lsmod | grep -E 'overlay|br_netfilter'
sysctl net.bridge.bridge-nf-call-iptables
sysctl net.bridge.bridge-nf-call-ip6tables
sysctl net.ipv4.ip_forward
```

- `overlay` 是 containerd 常用 OverlayFS 存储驱动依赖；
- `br_netfilter` 和 bridge sysctl 让 Pod 网络流量进入内核转发和过滤链路；
- `net.ipv4.ip_forward=1` 是跨节点 Pod 通信的基础。
