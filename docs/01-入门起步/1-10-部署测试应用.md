# 部署测试应用

本节创建一个 nginx Deployment，并通过 NodePort 暴露服务，用来验证集群调度、网络和 Service 转发能力。

## 创建应用

```bash
kubectl create deployment nginx --image=nginx:latest
kubectl expose deployment nginx --port=80 --type=NodePort
```

查看状态：

```bash
kubectl get deploy,pod,svc -o wide
```

确认：

- Deployment 有期望副本数。
- Pod 状态为 `Running`。
- Service 类型为 `NodePort`。
- Service 获得了 `30000-32767` 范围内的端口。

## 访问测试

获取 NodePort：

```bash
NODE_PORT=$(kubectl get svc nginx -o jsonpath='{.spec.ports[0].nodePort}')
curl "http://127.0.0.1:${NODE_PORT}"
```

如果从集群外访问，使用任意节点 IP：

```bash
curl "http://<node-ip>:${NODE_PORT}"
```

## 访问失败排查

先看资源状态：

```bash
kubectl get deploy,pod,svc -o wide
kubectl describe pod -l app=nginx
kubectl describe svc nginx
```

再看节点和网络：

```bash
kubectl get nodes -o wide
kubectl get pods -A -o wide
sudo ss -lntup
```

常见原因：

- Pod 镜像拉取失败。
- 节点没有 Ready。
- Calico 未正常运行。
- 防火墙没有放通 NodePort。
- 从外部访问时使用了错误的节点 IP。

## 清理资源

```bash
kubectl delete svc nginx
kubectl delete deployment nginx
```
