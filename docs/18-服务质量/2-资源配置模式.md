# 资源配置模式

资源配置首先服务于容量估算与稳定运行，QoS 是随之产生的分类。关键服务是否使用 Guaranteed，应基于可预测的 CPU、内存工作集和节点容量，而不是为了获得一个类别名称。

## Guaranteed 示例

```yaml [guaranteed-api.yaml]
apiVersion: v1
kind: Pod
metadata:
  name: guaranteed-api
spec:
  containers:
    - name: api
      image: nginx:1.31-alpine
      resources:
        requests:
          cpu: 250m
          memory: 256Mi
        limits:
          cpu: 250m
          memory: 256Mi
```

该 Pod 的唯一容器对 CPU 与内存均满足 request 等于 limit，因此 QoS 为 Guaranteed。CPU limit 会限制可用 CPU 时间；内存 limit 是 cgroup 上限，不应按瞬时峰值随意设置得过低。

## Burstable 示例

```yaml [burstable-worker.yaml]
apiVersion: v1
kind: Pod
metadata:
  name: burstable-worker
spec:
  containers:
    - name: worker
      image: alpine:3.23
      command: ["sh", "-c", "sleep 3600"]
      resources:
        requests:
          cpu: 100m
          memory: 128Mi
        limits:
          cpu: "1"
          memory: 512Mi
```

它预留基线资源并允许一定突发，适合资源需求有可测区间的普通服务。实际数值应来自观测数据并保留扩容与故障恢复余量。
