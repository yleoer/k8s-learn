# 设计思想

第 05 章完成了容器运行时和 CRI 的系统梳理，容器已经具备在单节点上稳定运行的基础。进入 Kubernetes 阶段后，学习重点从单节点容器运行转向多节点集群编排。

本章围绕 Kubernetes 的定位、架构、组件和核心资源展开，建立从“容器如何运行”到“集群如何编排”的整体认知。这些概念为后续学习 kubectl、Pod、Deployment、Service 等资源提供思维框架。

本章涵盖以下内容：

- Kubernetes 基本定位
- 云原生编排价值
- 集群架构全景
- 控制节点组件
- 工作节点组件
- 核心资源抽象
