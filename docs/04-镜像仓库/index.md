# 镜像仓库

第 03 章已经完成 Dockerfile、镜像优化和业务镜像制作的学习。镜像制作完成后，需要统一存储、分发和管理，才能被 Kubernetes、CI/CD 以及不同环境稳定拉取。

本章从镜像仓库概念出发，围绕 Harbor 安装部署、镜像推拉、客户端信任、权限控制和运维管理展开。完成本章后，后续 Kubernetes 私有镜像拉取、应用发布和企业交付流程才具备稳定镜像来源。

本章涵盖以下内容：

- 镜像仓库概念与地址规范
- Harbor 安装部署与基础配置
- Harbor 镜像推送与拉取
- Docker 与 containerd 客户端信任
- Harbor 用户、角色与项目权限
- Kubernetes 私有镜像拉取凭据
- 镜像清理、复制与运维管理
