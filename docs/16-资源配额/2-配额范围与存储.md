# 配额范围与存储

`scopes` 将配额统计限定到匹配条件的资源。常见范围包括 `BestEffort`、`NotBestEffort`、`Terminating`、`NotTerminating` 和 `PriorityClass`；不同 scope 支持的资源键不同，组合前应以 API 文档校验。

## 按优先级限制

```yaml [critical-quota.yaml]
apiVersion: v1
kind: ResourceQuota
metadata:
  name: critical-quota
  namespace: team-a
spec:
  scopeSelector:
    matchExpressions:
      - operator: In
        scopeName: PriorityClass
        values:
          - production-critical
  hard:
    pods: "6"
    requests.cpu: "2"
    requests.memory: 4Gi
```

该配额只统计引用 `production-critical` PriorityClass 的 Pod。优先级本身并不授予资源，仍需配合节点容量、调度规则和准入治理。

## 存储配额

`requests.storage` 限制 PVC 请求容量总和；也可使用 `<storage-class-name>.storageclass.storage.k8s.io/requests.storage` 约束特定 StorageClass。配额不保证后端存储一定可供给，PVC 仍可能因 CSI 驱动、拓扑或存储容量不足而停在 `Pending`。
