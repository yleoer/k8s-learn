# 节点维护与故障

维护窗口和节点故障属于不同的控制路径。计划维护应使用 `kubectl drain`，让驱逐 API、PodDisruptionBudget 和控制器参与迁移；节点异常由 Node lifecycle controller 添加内置污点并处理驱逐。

## 计划维护

```bash
kubectl cordon <node-name>
kubectl drain <node-name> --ignore-daemonsets --delete-emptydir-data
kubectl uncordon <node-name>
```

`cordon` 只阻止普通调度，不会驱逐已有 Pod。`drain` 前应核对 StatefulSet、持久卷、本地临时数据和 PDB；`--delete-emptydir-data` 会删除 `emptyDir` 数据，不能在未确认数据可丢失时使用。

## 内置污点

节点控制器会管理 `node.kubernetes.io/not-ready` 和 `node.kubernetes.io/unreachable` 等污点。Pod 对这两项 `NoExecute` 污点的默认容忍时间通常为 300 秒，实际 Pod 规范、控制器注入和集群配置仍应通过 `kubectl get po <pod-name> -o yaml` 核对。

不应为了“快速恢复”在所有工作负载中盲目缩短这些容忍时间。短容忍会更快创建替代副本，也会增加短暂网络波动期间的重调度、数据恢复和级联压力。
