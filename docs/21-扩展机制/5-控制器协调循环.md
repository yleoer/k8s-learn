# 控制器协调循环

控制器通过 informer 或等价机制观察对象变化，将键放入队列，并在 reconciliation 中读取最新状态、计算差异、创建或更新受管资源，最后更新 status。协调必须可重试且幂等，因为同一对象会被重复入队，事件可能合并或丢失。

## 协调步骤

```text
read custom resource
  -> validate supported intent
  -> observe owned resources and external state
  -> create or patch toward desired state
  -> update status and conditions
  -> requeue only when a later observation is needed
```

控制器应使用 ownerReferences 管理同命名空间的受管资源，使垃圾回收器能在自定义资源删除后清理它们。跨命名空间或集群外资源不能简单依赖 ownerReference，必须定义显式生命周期和清理策略。

## 并发与冲突

控制器写对象时要处理 `resourceVersion` 冲突，读取后重新计算并重试。不要在 reconcile 中依赖一次性内存状态，也不要每次事件都无条件写入对象，否则会造成热循环和 API Server 压力。
