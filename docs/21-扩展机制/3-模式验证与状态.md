# 模式验证与状态

CRD 的 OpenAPI v3 schema 负责对象结构、类型、枚举、范围和必填字段验证。应为 `spec` 建立结构化 schema，以便 API Server 校验、字段裁剪、OpenAPI 发布和客户端生成；避免无约束的自由 JSON 作为长期 API。

## Spec 与 Status

`spec` 表示用户期望状态，控制器读取它并执行协调；`status` 记录观察到的状态，例如 `observedGeneration`、条件、已就绪副本和最近错误。控制器写入 status 时应启用并使用 `status` 子资源，这样用户更新 spec 与控制器更新 status 可以按 API 边界分离。

下面片段仅说明条件字段关系，并不是完整 CRD：

```yaml{1-12}
status:
  observedGeneration: 3
  conditions:
    - type: Ready
      status: "True"
      reason: Reconciled
      message: Desired resources are available
      lastTransitionTime: "2026-07-15T00:00:00Z"
```

`metadata.generation` 在 spec 变更时递增。控制器应在成功处理该版本后更新 `status.observedGeneration`，让用户能判断当前状态是否反映最新规格。
