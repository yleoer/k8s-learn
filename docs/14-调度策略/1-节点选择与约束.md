# 节点选择与约束

调度器先过滤不满足硬约束的节点，再对可选节点打分。`nodeSelector` 适合一个确定的等值匹配；`nodeAffinity` 支持集合、存在性和软偏好。`nodeName` 会绕过调度器，通常只用于调试或由专用控制器明确指定节点，不应用于常规工作负载。

节点标签是放置策略的输入，不是安全边界。若标签决定隔离或受限硬件访问，Node authorizer 和 NodeRestriction 准入插件应处于正常工作状态，并为受保护标签使用 `node-restriction.kubernetes.io/` 前缀。

## 最小节点选择

下面的完整 Deployment 仅放到标记为 `workload-tier=general` 的节点。

```yaml [general-web.yaml]
apiVersion: apps/v1
kind: Deployment
metadata:
  name: general-web
spec:
  replicas: 2
  selector:
    matchLabels:
      app.kubernetes.io/name: general-web
  template:
    metadata:
      labels:
        app.kubernetes.io/name: general-web
    spec:
      nodeSelector:
        workload-tier: general
      containers:
        - name: web
          image: nginx:1.31-alpine
          ports:
            - containerPort: 80
```

```bash
kubectl label no <node-name> workload-tier=general
kubectl create -f general-web.yaml
kubectl get po -l app.kubernetes.io/name=general-web -o wide
```

当标签不存在或没有可用节点时，Pod 保持 `Pending`。应通过 `kubectl describe po <pod-name>` 查看 `FailedScheduling` 事件，而不是用 `nodeName` 强行绕开约束。

## 节点亲和性

`requiredDuringSchedulingIgnoredDuringExecution` 中的多个 `nodeSelectorTerms` 是“或”关系；一个 term 内的多个 `matchExpressions` 是“与”关系。`preferredDuringSchedulingIgnoredDuringExecution` 的每条偏好设置 1 到 100 的权重，调度器会将亲和性分数与其他评分插件合并。

```yaml{13-28} [analytics-worker.yaml]
apiVersion: apps/v1
kind: Deployment
metadata:
  name: analytics-worker
spec:
  replicas: 2
  selector:
    matchLabels:
      app.kubernetes.io/name: analytics-worker
  template:
    metadata:
      labels:
        app.kubernetes.io/name: analytics-worker
    spec:
      affinity:
        nodeAffinity:
          requiredDuringSchedulingIgnoredDuringExecution:
            nodeSelectorTerms:
              - matchExpressions:
                  - key: nodepool.example.com/class
                    operator: In
                    values:
                      - compute
          preferredDuringSchedulingIgnoredDuringExecution:
            - weight: 80
              preference:
                matchExpressions:
                  - key: topology.kubernetes.io/zone
                    operator: In
                    values:
                      - zone-a
      containers:
        - name: worker
          image: alpine:3.23
          command: ["sh", "-c", "while true; do sleep 3600; done"]
```

这里的 YAML 是完整资源，硬约束保证 Pod 只进入计算节点池，软偏好在其间优先选择 `zone-a`。节点不足时软偏好可以被放弃，硬约束不能。

> [!IMPORTANT]
> 亲和性只能影响新建 Pod 的调度。扩容、滚动更新和节点替换会重新判断规则，因此标签分类变更应纳入变更评审。
