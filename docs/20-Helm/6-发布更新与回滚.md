# 发布更新与回滚

Release 更新应从已审查的 Chart 版本和 values 文件发起。`helm upgrade --install` 可在 Release 不存在时安装，但它不应取代发布前对目标命名空间、现有资源和差异的确认。

## 受控更新

```bash
helm upgrade --install web ./web \
  --namespace team-a \
  --wait --timeout 5m
helm history web -n team-a
```

本地路径安装由 Chart.yaml 的 `version` 标识 Chart 版本。需要验证变更时，先在本地执行 `helm template web ./web --namespace team-a` 审查渲染结果；需要传入环境差异时，使用 [参数文件](/20-Helm/2-Chart结构与创建) 中的 `values.yaml` 作为基础并提供经过审查的覆盖文件。

对于 OCI Chart，版本固定在引用中：

```bash
helm upgrade --install web oci://registry.example.com/platform/web \
  --version 0.1.0 \
  --namespace team-a \
  --wait --timeout 5m
```

回滚使用历史修订号。它重新应用旧模板与值组合，仍可能受当前 API、准入策略、外部依赖和不可逆数据迁移限制。

```bash
helm rollback web <revision> -n team-a --wait --timeout 5m
```
