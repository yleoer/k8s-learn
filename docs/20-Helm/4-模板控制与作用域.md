# 模板控制与作用域

Helm 模板使用 Go template 语法和 Sprig 函数。`if` 控制是否渲染，`range` 遍历集合，`with` 改变当前作用域，管道符将前一表达式结果传给下一函数。过深的逻辑会降低 Chart 的可读性，应把复杂业务决策留在发布系统或生成 values 的上游。

## 条件渲染

下面是局部模板片段，仅说明 `if`、`with` 与 `nindent` 的配合：

```yaml{1-8} [templates/service.yaml]
{{- if .Values.service.enabled }}
apiVersion: v1
kind: Service
metadata:
  name: {{ include "web.fullname" . }}
  labels:
    {{- include "web.labels" . | nindent 4 }}
spec:
  type: {{ .Values.service.type }}
{{- end }}
```

```yaml{1-5} [templates/deployment-resources.yaml]
{{- with .Values.resources }}
resources:
  {{- toYaml . | nindent 2 }}
{{- end }}
```

`with` 块内的 `.` 变成 `resources` 对象，需要使用 `$` 访问根作用域。YAML 模板应在每次变更后用 `helm template` 与 YAML 校验工具检查，不能仅根据模板文本判断缩进正确。
