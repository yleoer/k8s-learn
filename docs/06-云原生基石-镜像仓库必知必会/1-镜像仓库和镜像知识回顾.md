# 03-1 镜像仓库和镜像知识回顾

镜像仓库 Registry 用于保存和分发容器镜像。开发、测试、生产环境中的镜像一般都会通过镜像仓库流转，而不是在每台机器上手工构建。

## 常用镜像仓库

| 镜像仓库 | 地址 | 说明 |
| --- | --- | --- |
| Harbor | <https://goharbor.io> | 企业常用私有镜像仓库，支持项目、用户、权限、复制、清理等能力 |
| 阿里云容器镜像服务 | <https://cr.console.aliyun.com> | 云上托管镜像仓库，适合国内网络环境和云上集群 |
| Docker Hub | <https://hub.docker.com> | Docker 官方公共镜像仓库，常用于获取基础镜像 |

## 镜像名称格式

完整镜像地址通常由仓库地址、命名空间或项目、镜像名和标签组成：

```text
registry.example.com/project/app:v1.0.0
```

示例：

```text
harbor.example.com/base/nginx:1.27
registry.cn-hangzhou.aliyuncs.com/demo/app:v1.0.0
nginx:alpine
```

如果没有显式指定仓库地址，Docker 默认从 Docker Hub 拉取。

## 常见操作流程

拉取镜像：

```bash
docker pull nginx:alpine
```

给镜像打标签：

```bash
docker tag nginx:alpine harbor.example.com/base/nginx:alpine
```

登录镜像仓库：

```bash
docker login harbor.example.com
```

推送镜像：

```bash
docker push harbor.example.com/base/nginx:alpine
```

## 私有仓库的价值

- 统一保存业务镜像，避免镜像散落在不同节点。
- 支持权限控制，区分开发、测试、生产镜像访问范围。
- 支持内网拉取，减少公网依赖。
- 支持镜像扫描、复制、清理和生命周期管理。
- 便于 Kubernetes、CI/CD 和制品管理平台集成。
