# 03-2 阿里云镜像仓库介绍及使用

阿里云容器镜像服务 ACR 可以托管容器镜像，常用于国内网络环境下的镜像存储、加速拉取和云上 Kubernetes 集群集成。

控制台地址：

```text
https://cr.console.aliyun.com
```

## 基本使用流程

使用阿里云镜像仓库前，一般需要完成：

1. 开通容器镜像服务。
2. 创建命名空间。
3. 创建镜像仓库。
4. 设置登录密码。
5. 使用 Docker 登录、打标签、推送镜像。

## 登录仓库

登录地址以实际地域为准，例如杭州地域：

```bash
docker login registry.cn-hangzhou.aliyuncs.com
```

登录时输入阿里云账号或 RAM 用户，以及容器镜像服务中配置的密码。

## 推送镜像

先准备一个本地镜像：

```bash
docker pull nginx:alpine
```

打标签：

```bash
docker tag nginx:alpine registry.cn-hangzhou.aliyuncs.com/YOUR_NAMESPACE/nginx:alpine
```

推送到阿里云镜像仓库：

```bash
docker push registry.cn-hangzhou.aliyuncs.com/YOUR_NAMESPACE/nginx:alpine
```

## 拉取镜像

```bash
docker pull registry.cn-hangzhou.aliyuncs.com/YOUR_NAMESPACE/nginx:alpine
```

如果仓库是私有仓库，拉取前需要先登录，或者在 Kubernetes 中配置 `imagePullSecrets`。

## 使用建议

- 命名空间按团队或业务线划分。
- 镜像仓库按应用划分，不要把多个无关应用混在同一个仓库。
- 镜像标签使用明确版本，例如 `v1.0.0`、`20260614-commitid`。
- 生产环境避免长期使用 `latest`。
- CI/CD 流水线中使用 RAM 用户或访问凭证，不要使用个人主账号。
