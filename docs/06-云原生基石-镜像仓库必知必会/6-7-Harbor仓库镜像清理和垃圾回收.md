# 03-7 Harbor 仓库镜像清理和垃圾回收

镜像仓库使用一段时间后会积累大量历史镜像。只删除镜像标签不一定立即释放磁盘空间，还需要结合保留策略和垃圾回收处理。

## 镜像清理的目标

- 删除无用的历史版本。
- 保留稳定版本和生产版本。
- 控制 Harbor 数据目录磁盘占用。
- 避免 CI/CD 高频推送导致仓库快速膨胀。

## 标签保留策略

Harbor 支持按项目配置标签保留策略，常见规则：

- 每个仓库保留最近 N 个镜像。
- 保留匹配版本规则的标签，例如 `v*`。
- 清理临时构建标签，例如 `dev-*`、`test-*`。
- 保留最近一段时间内推送的镜像。

示例策略：

```text
保留最近 10 个镜像标签
保留所有 v 开头的正式版本
清理 30 天前的临时构建镜像
```

## 手动删除镜像

在 Harbor Web 控制台中进入：

```text
项目 -> 镜像仓库 -> 选择镜像 -> 删除指定 tag 或 artifact
```

删除前要确认该镜像没有被 Kubernetes、发布系统或回滚流程继续使用。

## 垃圾回收

删除标签后，底层 blob 数据可能不会立刻释放。需要执行垃圾回收：

```text
系统管理 -> 垃圾回收 -> 立即执行
```

也可以配置定时垃圾回收，让 Harbor 定期释放无引用的数据。

## 清理注意事项

- 清理前确认生产环境和回滚版本仍可用。
- 不要只保留 `latest`，否则无法定位历史发布版本。
- CI/CD 推送镜像时要使用可追踪标签，例如版本号、构建号或 commit id。
- 垃圾回收可能影响仓库性能，建议安排在低峰期执行。
- Harbor 数据目录建议单独挂载磁盘，便于扩容和监控。

## 磁盘检查

查看 Harbor 数据目录占用：

```bash
du -sh /data/harbor
```

查看磁盘空间：

```bash
df -h
```
